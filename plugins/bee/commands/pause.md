---
description: Pause work and create a handoff file for seamless resumption
argument-hint: ""
---

## Current State (load before proceeding)

Read these files using the Read tool:
- `.bee/STATE.md` -- if not found: NOT_INITIALIZED
- `.bee/config.json` -- if not found: use `{}`

## Instructions

You are running `/bee:pause` -- an explicit pause command that captures current work state into a handoff file so you can resume seamlessly later. Follow these steps in order.

### Not Initialized

If the state above contains `NOT_INITIALIZED`, respond:

"BeeDev is not initialized. Run `/bee:init` first."

Stop here -- do not proceed.

### Step 1: Gather State

**From STATE.md**, read and extract:
- Current Spec: name, path, status (from the Current Spec section)
- Active phase: number, name, status (from the Phases table -- find the row that is not "Complete")
- Completed phases: list from Phases table where all status columns show completion
- Last Action: command, timestamp, result
- Quick Tasks: latest entry if any
- Decisions Log: any entries

**From git** (run via Bash):
- `git branch --show-current` -- current branch name
- `git status --short | wc -l` -- count of uncommitted files

**From TASKS.md** -- if an active phase exists, use Glob to find TASKS.md: `{spec-path}/phases/{NN}-*/TASKS.md` where NN is the zero-padded active phase number. If found:
- Read the TASKS.md file
- Completed tasks: lines matching `- [x]`
- Remaining tasks: lines matching `- [ ]`
- Failed tasks: lines matching `[FAILED]`
- Any task notes or decisions

### Step 2: Determine Next Action

Use the same next-command logic as `/bee:progress`:

| Current State | Suggested Command | Reason |
|--------------|-------------------|--------|
| No spec but uncommitted changes | `/bee:review-implementation` | "Uncommitted work exists. Review it first." |
| No spec (no uncommitted changes) | `/bee:new-spec` | "Start by defining what you want to build." |
| Spec exists, no phases planned | `/bee:plan-phase 1` | "Your spec is ready. Plan the first phase." |
| Next phase PENDING (not yet planned) | `/bee:plan-phase N` | "Phase N needs planning before execution." |
| Phase EXECUTING (mid-execution) | `/bee:execute-phase N` | "Phase N was interrupted. Resume execution." |
| Phase planned, not executed | `/bee:execute-phase N` | "Phase N is planned and ready to execute." |
| Phase executed, not reviewed | `/bee:review` | "Phase N is implemented. Time to review." |
| Phase reviewed, not tested | `/bee:test` | "Review is done. Generate test scenarios." |
| Phase tested, not committed | `/bee:commit` | "Tests pass. Ready to commit this phase." |
| All phases done | `/bee:complete-spec` | "All phases complete. Run the spec completion ceremony." |
| Status is ARCHIVED | `/bee:new-spec` | "Previous spec archived. Start a new one." |

### Step 3: Write Handoff File

Determine the per-spec handoff path: `.bee/specs/<slug>/pause-handoff.md` where `<slug>` is the Current Spec Path slug from STATE.md (e.g., if Path is `.bee/specs/my-feature/`, slug is `my-feature`). If NO_SPEC (no active spec), fall back to `.bee/pause-handoff.md` for the global path.

Write the handoff to the per-spec path (or global fallback if no spec) using the Write tool with this structure:

```markdown
---
paused_at: {ISO 8601 timestamp}
branch: {current git branch}
uncommitted: {count of uncommitted files}
spec_slug: {Current Spec Path slug from STATE.md, e.g. "my-feature" — or empty string if NO_SPEC}
---

# Pause Handoff

> Auto-generated by /bee:pause on {YYYY-MM-DD HH:mm}

## Current Position
- **Spec:** {spec name} ({status}) -- or "No active spec"
- **Spec Slug:** {Current Spec Path slug from STATE.md} -- or "(none)"
- **Phase:** Phase {N} -- {phase name} ({phase status}) -- or "No active phase"
- **Stack:** {stack from config.json}

## Completed Work
{List completed phases as "- Phase {N}: {name} -- complete"}
{List completed tasks from TASKS.md as "- [x] {task description}"}
{If nothing: "Nothing completed yet."}

## Remaining Work
{List remaining phases as "- Phase {N}: {name} -- {status}"}
{List remaining tasks from TASKS.md as "- [ ] {task description}"}
{If nothing: "All work complete."}

## Blockers
{List any [FAILED] tasks as "- {task}: {failure reason}"}
{If none: "None"}

## Decisions Made
{List Decisions Log entries as "- {decision}"}
{If none: "None recorded this session"}

## Next Action
**Run:** {suggested next command}
**Why:** {brief reason from Step 2}
```

### Step 3.5: Update STATE.md Last Action

Re-read `.bee/STATE.md` from disk (Read-Modify-Write pattern -- always read the current version before writing to avoid stale overwrites).

Update Last Action section:
- Command: `/bee:pause`
- Timestamp: current ISO 8601 timestamp
- Result: "Work paused. Handoff saved to {per-spec handoff path}" (use `.bee/specs/<slug>/pause-handoff.md` if spec active, or `.bee/pause-handoff.md` if no spec)

### Step 4: Confirm

Display to the user:

```
Work paused. Handoff saved to .bee/specs/<slug>/pause-handoff.md

Position: {spec name} > Phase {N} ({status})
Branch: {branch} | Uncommitted: {count} files
Next action: {suggested command} -- {reason}

Resume later with /bee:resume -- it will detect .bee/pause-handoff.md automatically.
```

Then present an interactive menu:

```
AskUserQuestion(
  question: "Pause complete. Handoff saved.",
  options: ["Compact now", "Custom"]
)
```

- **Compact now**: Display "Run `/compact` to compress the conversation." Do NOT attempt to invoke /compact programmatically.
- **Custom**: Wait for free-text input.
